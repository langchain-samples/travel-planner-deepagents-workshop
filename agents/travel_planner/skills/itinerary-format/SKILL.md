# Itinerary Format

Use this exact structure when writing `/itinerary.md`:

```
# <Destination> -- <Trip Dates>

## Trip Overview
- Travelers: ...
- Style: ...
- Budget: ...

## Day 1 -- <Date> (<Day of week>)
- Morning:   ...
- Afternoon: ...
- Evening:   ...
- Lodging:   ...

## Day 2 -- <Date> ...
(repeat per day)

## Logistics
- Flights:    ...
- Transit:    ...
- Currency:   ...
```

Always include time blocks (Morning/Afternoon/Evening). Always end with a Logistics section.
